# Plan de Reparación y Mejora del Código - WoldVirtual3D Viewer

## 1. Estado Actual
El proyecto se encuentra funcional pero contiene deuda técnica y código legado que debe ser limpiado para asegurar la estabilidad y mantenibilidad futura. Se han resuelto recientemente problemas críticos de compilación (duplicidad de clases) y funcionalidad (foco de teclado en el juego).

## 2. Tareas de Refactorización Prioritarias

### A. Limpieza de `GodotService.cs`
- **Problema**: Contiene métodos obsoletos y lógica de "fuerza bruta" para encontrar rutas.
- **Acciones**:
  1. Eliminar el método `LaunchGodotSceneAsync` (marcado actualmente como `[Obsolete]`) una vez que todos los consumidores (LoginViewModel) se hayan actualizado.
  2. Centralizar la configuración de rutas (`godot_path.txt`) en un `ConfigurationService` robusto, eliminando las rutas "hardcoded" (`D:\woldvirtual3d...`) del código fuente.
  3. Limpiar los `DllImport` no utilizados si los hay.

### B. Actualización de `LoginViewModel.cs`
- **Problema**: Utiliza el método obsoleto `LaunchGodotSceneAsync`.
- **Acciones**:
  1. Actualizar `LaunchGodotAsync` para usar `LaunchGodotForEmbeddingAsync`.
  2. Asegurar que el manejo del proceso (Process) se transfiera correctamente al `GameViewModel` o se gestione centralizadamente.

### C. Consolidación de `GameViewModel.cs`
- **Estado**: Recientemente refactorizado para thread-safety y manejo de foco.
- **Acciones**:
  1. Verificar que `RunOnUIThread` cubra todas las actualizaciones de `ObservableCollection`.
  2. Eliminar código muerto o comentado si existe.

### D. Gestión de Foco (Win32 Interop)
- **Estado**: Implementado sistema híbrido WPF/Win32 (`SetFocus`, `FocusGame`).
- **Acciones**:
  1. Documentar claramente el flujo de foco entre Overlay (WPF) y Juego (Godot/Win32) para futuros desarrolladores.
  2. Mantener la lógica de "Click-through" en `GameOverlayWindow` (Background null) para evitar bloquear el juego.

## 3. Plan de Ejecución
1. **Fase 1 (Completada)**: Estabilizar compilación y arreglar input de usuario (Flechas/Espacio).
2. **Fase 2 (Inmediata)**: Actualizar `LoginViewModel` para eliminar warnings de métodos obsoletos.
3. **Fase 3**: Refactorizar `GodotService` para limpieza de rutas.

## 4. Notas Técnicas
- **Compilación**: Ejecutar siempre `dotnet clean` tras refactorizaciones grandes de clases para limpiar cachés de obj/bin.
- **Dependencias**: Mantener `System.Management` y `Newtonsoft.Json` actualizados.
